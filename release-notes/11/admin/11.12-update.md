# Service 11.12 update

## Requests - Add Relation rules

You can create hierarchical and side‑by‑side relationships (Related, Parent or Child), helping you understand dependencies, track progress and manage complex service scenarios with confidence. This gives you better control, improved visibility and a smoother way to work with interconnected requests.

Read more about [how to use Related requests][2] in SuperOffice Service.

There are three default relations types that cannot be edited or deleted, but you can add additional request rules to support your workflow in SuperOffice Service.

![Admin Requests - Add a new request rule and add the correct condition and behavior  -screenshot][img1]

When creating a new relation rule, you can add conditions From (source) and To (target) of Request types with a specific Relation type. You also add different behavior of the rule like "require parent to be set" and "Prevent parent closure". The rule triggers automatic actions in the request user interface only.
