# Service 11.11 update

## Requests - Created date to use in columns and criteria

To get that overview of number of requests per day, in either selections or in dashboard, it's a good idea to use the field **Created date** that only search for the date and not the time as well (as the **Created** field does).

![The request messages show the same time in preview and in the list –screenshot][img1]

You can use this **Created date** field both as a column and as a criteria. As a criteria you see there is a difference between the two created criteria, where you see a calendar icon on the Created date criteria and a calendar icon + a clock icon on the Created criteria.

![The request messages show the same time in preview and in the list –screenshot][img2]
