# Service 12.0 update

## SPAM classification

SuperOffice now tracks the number of resolved support requests per account as the basis for usage-based billing. The count reflects requests closed across all channels - email, chat, phone, web portal, and API - minus any requests marked as SPAM.

A new **Mark as SPAM** action lets administrators classify fraudulent or unwanted requests. SPAM requests are excluded from all views, searches, reports, and metering - they will not inflate your usage count.

* Only administrators and users with the Can mark requests as SPAM right can classify requests.
* Marked requests move to a dedicated SPAM recycle bin, where you can restore them if needed.
* After a configurable retention period, SPAM requests are permanently deleted.

> [!NOTE]
> You need the functional right "Can mark requests as Spam" to be able use this function.
