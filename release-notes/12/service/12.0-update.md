# Service 12.0 update

## SPAM classification on requests

SuperOffice tracks the number of resolved support requests per account as the basis for usage-based billing. The count reflects requests closed across all channels - email, chat, phone, web portal, and API - minus any requests marked as Spam.

The **Mark as SPAM** action lets users classify fraudulent or unwanted requests. SPAM requests are excluded from all views, searches, reports, and metering - they will not inflate your usage count.

* Only users with the **Can mark requests as Spam** right can classify requests. The functional right **Can mark requests as Spam** is set for Roles in Settings and maintenance.
* Marked requests move to a dedicated SPAM recycle bin, where you can restore them if needed. You find the **Recycle bin** in your Personal settings menu. After a configurable period, Spam requests are permanently deleted.

![Request - Add relations between multiple requests –screenshot][img1]

You see **Mark as Spam** in the context menu for requests in your lists, under the Task menu on one request, and in the Task menu on request selections.

You will get a confirmation dialog (Yes/No) after selecting the menu item Mark as Spam, to **confirm** that you want to set the request(s) as Spam.
